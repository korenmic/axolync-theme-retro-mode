# Retro Theme Acceptance Notes

These notes are the repo-owned visual target for later browser Playwright proof.

## Shared Rules

- Treat the four retro themes as one family, but never let them collapse into palette swaps.
- Latin and Hebrew must both remain readable.
- Hebrew may soften into a hybrid retro treatment when strict segmentation becomes unreadable.
- Decorative surfaces may be ambitious, but karaoke timing truth must stay untouched.

## Retro Red LED

- Feels like a dark red dashboard or wall clock display.
- Segmented glyph energy and red glow should dominate.
- Panels should feel heavy, warm, and slightly smoked.
- The lyric active state should feel brighter and hotter than the idle state.

## Retro Cyan VFD

- Feels like a vacuum-fluorescent display panel.
- Cyan glow should be cool and bright, not blue-purple.
- Segmentation should be thinner and cleaner than the red LED theme.
- Shell surfaces should feel glassy and dark, not pixelated.

## Retro Arcade CRT

- Feels like an arcade cabinet or pixel-space game overlay.
- The shell should support visible pixel-art cues and CRT scanline softness.
- Accent colors should feel playful and game-like.
- Decorative imagery should reinforce the cabinet / starfield / game-screen energy.

## Retro Cockpit HUD

- Feels like a spacecraft or aircraft instrument display.
- Panels should look angular, framed, and engineered.
- Cyan HUD highlights should read as instrumentation, not VFD.
- Decorative assets should feel like non-clickable cockpit chrome.

## Screenshot-Proof Expectations

- Every screenshot should make the theme recognizable without reading the theme label.
- Lyrics, song title, artist, status, progress, and shell controls should all visually participate.
- Existing supported themes are still expected to remain valid after Retro Mode lands.
